# 문제

메인 화면은 완성도가 높은데 내부 페이지는 전혀 다른 스킨을 사용하고 있어서, 사용자가 탐색을 시작하는 순간 경험이 끊겼다.

# 설계 방향

홈과 내부 페이지를 한 시스템처럼 보이게 만들려면 다음 세 가지가 먼저 정리되어야 했다.

1. 홈에서 어떤 분류로 사용자를 보낼지
2. 그 분류가 실제 URL과 템플릿으로 어떻게 연결될지
3. 단일 글과 목록 페이지가 어떤 공통 시각 언어를 공유할지

# 선택

`/troubleshooting/`, `/architecture/`, `/devlog/`처럼 목적이 분명한 허브 페이지를 두고, 실제 글은 기존 `posts`와 `projects` 안에 두되 태그와 카테고리로 모으는 구조를 택했다.

# 이유

콘텐츠 저장 위치를 크게 흔들지 않으면서도, 상위 내비게이션에서 의미 있는 이동 경로를 만들 수 있기 때문이다.
